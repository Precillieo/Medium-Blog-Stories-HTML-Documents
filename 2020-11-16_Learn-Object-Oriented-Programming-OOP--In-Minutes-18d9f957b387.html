What is Self?

Self stands for future objects.

Objects of a class don’t yet exist when a class is being defined. But we often need a way to refer to the data of a particular object within the class definition. It is a stand-in for a particular object within a class definition, also a reference to the current instance of the class, and it’s used to access variables that belong to the class.

Still, it’s neither a keyword nor special name but a conventional name which can be replaced with any other words like “kitty”, “ iris”, e.t.c based on the programmer’s intent as long as they’re used consistently without replacement.

Note: The self parameter should be the first argument of the regular instance method. Nevertheless, always keep self as self.

Inheritance

It refers to defining a new class with little or no modification of an existing class. Inheritance inherits all methods and properties from another class. It allows code reusability, allows classes to inherit features of other classes.

New Class Functionality = Old Class Functionality + extra

You know, for example, Cancer extends attributes and functionalities to Breast, Cervical, Lung, Skin Cancers respectively. Meaning, all Breast Cancers are Cancer but all Cancers are not Breast Cancer. Inheritance represents “is a” relationship.

Polymorphism

In a literal sense, polymorphism is the ability to assume different shapes and forms. In Python, it means a variable, function, or object can take on multiple forms. While in OOP, polymorphism allows us to define methods in both the child class and parent class with the same name. The work of polymorphism can be addressed in two different ways;

  • Method Overriding
  • Method Overloading

Method Overriding

This is used in situations where a method inherited by a child class from a parent class doesn’t fit in the child's class. Meaning, we’d have to re-implement the method in the child class.

Here, the parent class Dog has the unique ability to bark with a louder sound. Child class Puppy can make sounds as well but a less loud one. Significantly, the method inherited from the parent class wouldn’t fit in the child class, so a different implementation is provided for the child class.

Meanwhile, each of the methods is called based on the type of object used in calling it. When Puppy class object py is used to call an overridden method bark() then the Puppy class version of the method HOWL is printed. But if the Dog class object dg with the method bark() is called, WOOF is printed out.

Abstraction

Abstraction means hiding the complexity or internal implementation from the users.

  • Simple codes are used to represent complexity.
  • Hide complex details from the users.

A real-life example is the details of how the engine transmission and braking systems of a car works whose execution and automation are not known to the people driving the car. A car is not seen as different individual parts but a well-defined object with its distinct behaviors. Abstraction in OOP is achieved by using abstract classes and methods.

An abstract class provides incomplete functionality and contains one or more abstract methods. Abstract methods don’t have any implementation but the child classes are left to provide the implementation for the abstract methods.

Design Techniques Recommended for OOP Codes

  • Class names are to start with capital letters. If a class name contains more than one word, each should begin with a capital letter e.g ClassName. They’re seen to be nouns.
  • The pattern of functions and loops are expected. Indentation as proper in the line of codes that come after the class.
  • The definition of a method must start with the def keyword.
  • Method names must begin with lowercase letters. If it’s more than one word, it should either have a hyphen (_) in between the words e.g method_name, or every word after the first begins with a capital letter e.g methodName().
  • Parameters should be as clear as possible indicating its purpose and existence.
  • Names should never start with special characters e.g &, %, _, €.
  • A string literal (‘’’) and not other type of expressions occurring immediately after a class or function at the top before any other statement is a docstring. This helps users understand the intuitions behind the classes, methods, and attributes. It can be seen as output by calling the python built-in help function on the specific class or method. In a scenario where many developers contribute to a certain project, it’s a way to pass information about classes and methods created to help others understand each of their written codes and ensure code reusability.
  • Finally, write good codes. Portability, maintainability, readability matter!!!
